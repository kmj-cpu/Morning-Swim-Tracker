# 🏊‍♂️ Morning-Swim-Tracker: 아침 수영 마일리지 분석기

> **"불규칙한 기상 시간과 수영 거리를 매일 기록하고, 이전 주와 비교해 증감률과 평균을 계산해 주는 프로그램입니다."**

---

## 🤔 기획 배경 및 목적
저는 매일 아침 **7시 수영 루틴**을 실천하고 있으며, **라이프가드 및 수상안전강사 자격증**을 보유할 정도로 수영에 대한 열정이 강합니다. 

하지만 의지만으로는 제어하기 힘든 '기상 시간'의 미세한 차이가 당일 수영 거리(500m~1000m)를 결정짓는 핵심 변수가 되고 있으며, 이를 체계적으로 관리하지 못해 운동 성과를 객관적으로 파악하기 어려운 실정입니다. 

> **"이에 저의 전공 분야인 통계적 관점을 접목하여 불규칙한 기상 패턴과 운동량 데이터를 누적 분석하고자 합니다."**
> 
> 단순한 기록을 넘어 **데이터 기반의 피드백**을 통해 수영 루틴을 단단히 정착시키고, 지속적인 성장을 위한 동기부여를 얻는 것이 본 프로젝트의 핵심 목적입니다.

---

## ✨ 핵심 기능 3가지

1. **📝 일일 기록 입력 및 출력**
   * 당일의 기상 시간과 수영 거리(m)를 입력받습니다.
   * `기록 완료! 오늘 {time}에 일어나서 {distance}m를 수영했습니다.` 와 같이 직관적인 확인 메시지를 출력합니다.

2. **💾 주간 통계 연산 및 영구 저장**
   * 입력받은 기록들을 누적하여 평균 기상 시간과 수영 거리 합계를 계산합니다.
   * 프로그램을 껐다 켜도 데이터가 유지되도록 파일 형태로 저장하고 불러옵니다.

3. **📈 이전 주 대비 증감률 비교**
   * 파일에 저장된 이전 주 데이터와 이번 주 데이터를 대조하여 수영 거리 증감률(%)을 계산합니다.

## 🛠️ 사용할 Python 핵심 개념

* **🔀 변수와 제어문**
  기상 시간과 수영 거리를 입력받아 변수에 할당하고, `for`문을 이용해 이전 기록들을 순회하며 합계와 평균을 계산합니다.
* **📦 리스트와 딕셔너리**
  오늘 하루의 단위 기록(날짜, 기상 시간, 수영 거리)을 `딕셔너리(Dictionary)`로 묶고, 이를 전체 `리스트(List)`에 차곡차곡 누적 저장하여 데이터를 구조화합니다.
* **⚙️ 함수**
  데이터 입력을 받는 부분, 통계를 연산하는 부분 등을 각각의 `함수(Function)`로 분리하여 코드의 가독성과 유지보수성을 높입니다.
* **📂 파일 입출력**
  기록된 리스트 데이터를 `.txt`나 `.csv` 파일로 내보내고, 프로그램을 다시 켤 때 기존 데이터를 불러오는 기능을 추가로 학습하여 구현합니다.

## 🚀 최종 코드

```python
#아침 수영 마일리지 분석기 찐 최종 코드
records = []
#프로그램 켤 때 기존 파일 읽어오기 ('r' 모드) ---
try:
    file = open("swim_data.txt", "r", encoding="utf-8")
    for line in file:
        # 줄바꿈(\n)을 없애고 쉼표(,)를 기준으로 데이터를 쪼갭니다.
        # 예: "2026-06-11,06:00,1000" -> ['2026-06-11', '06:00', '1000']
        data = line.strip().split(",")

        # 쪼갠 데이터를 다시 딕셔너리로 조립해서 리스트에 복구합니다.
        recovered_record = {
            "date": data[0],
            "wake_time": data[1],
            "distance": int(data[2]) # 거리는 계산을 위해 다시 숫자로 변환
        }
        records.append(recovered_record)
    file.close()
except FileNotFoundError:
    # 맨 처음 실행해서 아직 텍스트 파일이 없을 때는 그냥 넘어갑니다.
    pass

while True:
    print("\n---아침 수영 마일리지 ---")
    print("1. 오늘 기록 입력하기")
    print("2. 주간 통계 확인하기")
    print("3. 프로그램 종료")

    choice = input("메뉴를 선택하세요 (1/2/3): ")

    if choice == '1':
        print("\n>> 기록 입력 화면입니다.")
        date = input("오늘 날짜를 입력하세요 (예: 2026-06-11): ")
        wake_time = input("오늘 기상 시간을 입력하세요 (예: 06:00): ")
        distance = int(input("오늘 수영한 거리를 입력하세요 (m): "))

        daily_record = {
            "date": date,
            "wake_time": wake_time,
            "distance": distance
        }
        records.append(daily_record)

        # 불러오기 쉽게 쉼표(,)로만 구분해서 저장합니다.
        file = open("swim_data.txt", "a", encoding="utf-8")
        file.write(f"{date},{wake_time},{distance}\n")
        file.close()

        print(f">> 기록 완료! 오늘 {wake_time}에 일어나서 {distance}m를 수영했습니다.")

    elif choice == '2':
        print("\n>> 주간 통계 화면입니다.")

        if len(records) == 0:
            print("아직 입력된 기록이 없습니다. 1번 메뉴에서 기록을 먼저 입력해주세요.")
        else:
            total_distance = 0
            for record in records:
                total_distance = total_distance + record["distance"]

            print("\n--- 이번 주 수영 통계 ---")
            print(f"* 기록된 날짜 수: {len(records)}일")
            print(f"* 누적 총 수영 거리: {total_distance}m")

            if total_distance >= 20000:
                print("* 🧜‍♂️ 누적 20,000m 달성! 엄청난 갓생 루틴을 증명하셨습니다!")
            elif total_distance >= 5000:
                print("* 이번 주 기준치 5,000m 돌파! 달콤한 에그타르트 하나 드시면서 푹 쉬세요!")
            else:
                print("* 꾸준한 아침 루틴! 내일 새벽 수영도 화이팅입니다.")

    elif choice == '3':
        print("\n프로그램을 종료합니다. 내일도 상쾌한 물살 가르시길 바랄게요!")
        break

    else:
        print("\n잘못된 입력입니다. 1, 2, 3 중에서 다시 선택해주세요.")
```
## AI 도구 사용 내역

1. [Gemini]

* 어디에 사용: 데이터 영구 저장을 위한 파일 이어쓰기( `'a'` 모드) 방법 질문
* 받은 도움: `open("swim_data.txt", "a")` 를 사용하여 기존 내용을 지우지 않고 밑에 이어 쓰는 원리 학습
* 내 판단: 파일 입출력의 `'a'` 모드는 매일의 기록을 누적하려면 덮어쓰기(`'w'`)가 아닌 이어쓰기가 필수라고 판단하여 원리를 숙지한 후 적용함.

2. [Gemini]

* 어디에 사용: 프로그램을 껐다 켰을 때 과거 기록이 리스트에서 초기화되는 버그 해결
* 받은 도움: 프로그램 시작 시 `'r'` 모드로 파일을 읽어오고, `split(",")` 으로 문자열을 쪼개어 다시 딕셔너리와 리스트로 복구하는 로직, 그리고 파일이 없을 때를 대비한 `try-except` 예외 처리 학습
* 내 판단: 테스트 단계에서 데이터가 날아가는 치명적인 문제를 직접 발견함. 파일 읽기 로직과 예외 처리에 익숙하지 않으나 프로그램의 핵심 가치인 '누적 통계'를 위해 반드시 필요하다고 판단함. 복잡한 외부 라이브러리를 쓰지 않고 배운 지식(문자열 처리, 딕셔너리)을 응용하는 선에서 코드를 완성함.
